# X (Twitter) 登入 - 應用程式狀態檢查

> **建立日期**：2025-01-29  
> **目的**：檢查 X Developer Portal 應用程式狀態

---

## 🔍 如何檢查應用程式狀態

### 方法 1：在 X Developer Portal 主頁面

1. **登入 [X Developer Portal](https://developer.x.com/)**

2. **進入您的專案**：
   - 在左側選單中，點擊您的專案名稱
   - 或從主頁面選擇您的專案

3. **查看應用程式列表**：
   - 在專案頁面中，您會看到應用程式列表
   - 每個應用程式旁邊會顯示狀態標籤

4. **狀態標籤**：
   - **Active**（綠色）：應用程式已啟用，可以正常使用
   - **Suspended**（紅色或黃色）：應用程式已被暫停
   - **Pending**（黃色）：應用程式正在審核中
   - **Inactive**（灰色）：應用程式未啟用

---

### 方法 2：在應用程式設定頁面

1. **進入應用程式設定**：
   - 點擊應用程式名稱進入詳細頁面
   - 或進入 **「Settings」** → **「User authentication settings」**

2. **查看狀態資訊**：
   - 在頁面頂部或側邊欄中，通常會顯示應用程式狀態
   - 狀態會以標籤或文字形式顯示

---

### 方法 3：在 Keys and tokens 頁面

1. **進入 Keys and tokens**：
   - 在左側選單中，點擊 **「Keys and tokens」**

2. **查看狀態**：
   - 如果應用程式被暫停，通常會在頁面頂部顯示警告訊息
   - 或顯示狀態標籤

---

## 📋 狀態說明

### Active（啟用）

- ✅ **正常狀態**：應用程式可以正常使用
- ✅ **OAuth 功能正常**：可以進行第三方登入
- ✅ **API 調用正常**：可以調用 X API

**如果狀態是 Active**：
- 應用程式設定正確
- 問題可能在其他地方（如 Supabase 設定）

---

### Suspended（暫停）

- ❌ **應用程式已被暫停**：無法使用 OAuth 功能
- ❌ **可能的原因**：
  - 違反 X Developer Policy
  - 安全問題
  - 其他違規行為

**如果狀態是 Suspended**：
- 需要聯繫 X 支援解除暫停
- 或創建新的應用程式

---

### Pending（待審核）

- ⏳ **應用程式正在審核中**：暫時無法使用
- ⏳ **審核時間**：通常需要幾天到幾週

**如果狀態是 Pending**：
- 需要等待審核完成
- 或檢查是否有待處理的審核項目

---

### Inactive（未啟用）

- ⚠️ **應用程式未啟用**：需要啟用才能使用

**如果狀態是 Inactive**：
- 需要啟用應用程式
- 或檢查是否有未完成的設定

---

## 🔧 如果狀態不是 Active

### 如果是 Suspended

1. **查看暫停原因**：
   - 在應用程式頁面中，通常會顯示暫停原因
   - 或查看通知/郵件

2. **聯繫 X 支援**：
   - 前往 [X Developer Support](https://developer.x.com/en/portal/support)
   - 提交支援請求
   - 說明情況並請求解除暫停

3. **或創建新應用程式**：
   - 如果無法解除暫停，可以創建新的應用程式
   - 重新配置所有設定

---

### 如果是 Pending

1. **檢查審核狀態**：
   - 查看是否有待處理的審核項目
   - 確認所有必要資訊已填寫

2. **等待審核完成**：
   - 審核通常需要幾天到幾週
   - 可以查看審核進度

3. **或使用測試環境**：
   - 某些情況下可以使用測試環境
   - 但生產環境需要審核通過

---

### 如果是 Inactive

1. **啟用應用程式**：
   - 在應用程式設定頁面中，找到啟用選項
   - 點擊啟用

2. **完成必要設定**：
   - 確認所有必要欄位已填寫
   - 確認 Callback URI 等設定正確

---

## 🎯 下一步

### 如果狀態是 Active

**問題可能在其他地方**：

1. **檢查 Supabase Provider 狀態**：
   - 確認 X Provider 真的啟用
   - 確認憑證正確

2. **檢查 Supabase Site URL**：
   - 確認 Site URL 設定正確

3. **查看 Supabase Authentication Logs**：
   - 查看詳細的錯誤訊息
   - 了解 Supabase 為什麼拒絕請求

### 如果狀態不是 Active

**需要先解決狀態問題**：

1. **如果是 Suspended**：
   - 聯繫 X 支援
   - 或創建新應用程式

2. **如果是 Pending**：
   - 等待審核完成
   - 或檢查審核狀態

3. **如果是 Inactive**：
   - 啟用應用程式
   - 完成必要設定
