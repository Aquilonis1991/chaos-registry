如果頁面完全空白，可能是環境變數未正確設置。

### 1. 檢查 .env.local 檔案是否存在

在專案根目錄（與 `package.json` 同級）檢查是否有 `.env.local` 檔案。

**Windows PowerShell 檢查：**
```powershell
Test-Path .env.local
```

如果顯示 `False`，表示檔案不存在。

### 3. 重新啟動開發伺服器

**重要**：環境變數更改後必須重啟！

1. 在終端機按 `Ctrl+C` 停止當前伺服器
2. 重新執行：
```bash
npm run dev
```

### 4. 檢查瀏覽器 Console

1. 打開 http://localhost:8080/
2. 按 `F12` 打開開發者工具
3. 查看 Console 標籤

**如果環境變數正確設置**，應該看到：
- `Starting VoteChaos App...`
- `React rendered`
- 沒有錯誤訊息

**如果環境變數未設置**，應該看到：
- `❌ Supabase 環境變數未設置！`
- 頁面顯示錯誤訊息

## 常見問題

### Q1: 檔案名稱正確但還是空白？
**A**: 確認檔案名稱是 `.env.local`（開頭有點，沒有空格）

### Q2: 已創建檔案但還是空白？
**A**: 
1. 確認檔案內容格式正確（每行一個變數，沒有多餘空格）
2. 確認已重新啟動開發伺服器
3. 清除瀏覽器緩存（Ctrl+Shift+Delete）

### Q3: 如何確認環境變數已載入？
**A**: 在瀏覽器 Console 輸入：
```javascript
import.meta.env.VITE_SUPABASE_URL
```
應該顯示：`"https://epyykzxxglkjombvozhr.supabase.co"`
